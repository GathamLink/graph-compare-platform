## 1. 这个平台是什么

**图片对比平台**是一个帮助你快速发现"两组图片之间差异"的工具。

### 解决的问题

如果你有以下需求，这个平台正好适合你：

| 场景 | 举例 |
|------|------|
| 文档在不同平台导出，需要对比渲染效果 | 同一份 Word/PDF 在 Windows 和 macOS 导出，字体/间距是否一致？ |
| 网页/App 改版前后对比 | 页面改版后，哪些地方视觉上有变化？ |
| 图片批量审核 | 几十组图片，哪些有明显差异？哪些看起来一样？ |
| 设计稿与实现对比 | 设计师的稿子和开发实现的截图，差在哪里？ |

### 核心能力

- **自动配对**：上传两组图片后，自动按顺序一一配对（也支持按文件名前缀匹配）
- **量化评分**：每对图片给出 0~100% 的相似度分数
- **差异标注**：自动生成差异图，用红框标注出变化的区域
- **批量处理**：一次处理几十上百对图片，结果统一列表展示
- **报告导出**：一键导出 HTML 格式的对比报告，方便分享

### 使用者

- 产品经理：UI/功能改版前后对比
- 测试工程师：回归测试中的截图对比
- 内容/运营：文档导出质量检查
- 任何需要批量对比图片差异的人

## 2. 快速开始

### 2.1 本地安装（macOS / Linux）

> 整个安装过程约 5~10 分钟（取决于网速），只需执行两条命令。

**第一步：下载代码**

```bash
git clone https://github.com/GathamLink/graph-compare-platform.git
cd graph-compare-platform
```

**第二步：一键安装所有依赖**

```bash
bash scripts/install.sh
```

安装脚本会自动完成：
- 下载 MinIO（图片存储服务）
- 安装 Python 环境和后端依赖
- 安装 Node.js 和前端依赖
- 生成配置文件

**第三步：启动服务**

```bash
bash scripts/start.sh
```

启动成功后，打开浏览器访问：**http://localhost:13010**

### 2.2 第一次使用流程

下面用一个具体例子演示完整流程：

> **目标**：对比同一份文档在两个平台导出的图片，找出差异。

**Step 1：创建任务**

1. 打开平台首页，点击右上角「**+ 新建任务**」
2. 填写任务名称（如"文档导出对比-2026-04"）
3. 选择**配对方式**：
   - **顺序配对**：A 组第1张 ↔ B 组第1张（推荐，简单直接）
   - **前缀配对**：文件名以 `_A` / `_B` 结尾时自动匹配
4. 选择**对比算法**（不确定就选默认的「标准模式」）
5. 点击「创建」

**Step 2：上传图片**

平台进入任务管理页面后，有两种上传方式：

**方式 A：ZIP 批量导入（推荐，适合大量图片）**

将图片按以下结构打包成 ZIP：

```
你的文件名.zip
  └── 任意目录名/
      ├── 原图/        ← 放第一组图片
      │   ├── 001.png
      │   └── 002.png
      └── 对比图/      ← 放第二组图片
          ├── 001.png
          └── 002.png
```

> **macOS 打包方法**：新建文件夹，放入 `原图` 和 `对比图` 两个子文件夹，选中这两个子文件夹 → 右键 → 压缩

点击「**批量追加**」按钮，上传 ZIP 文件即可。

**方式 B：手动上传**

点击「图片组 A」或「图片组 B」的上传区域，直接拖拽或点击选择图片。

**Step 3：等待对比完成**

上传完成后，系统自动在后台计算差异。页面顶部会显示计算进度条，全部完成后进度条变绿。

**Step 4：查看结果**

点击「**开始对比**」或直接点击卡片进入**对比结果列表**：

- 🟢 **相同**：相似度 ≥ 75%，两张图基本一致
- 🔴 **差异**：相似度 < 75%，有明显不同
- 🟡 **异常**：计算失败或图片尺寸差异过大

点击任意一行，弹窗展示详细对比视图。

## 3. 功能说明

### 3.1 任务管理

任务是平台的基本管理单元，每个任务对应一组"两组图片的对比需求"。

| 操作 | 位置 | 说明 |
|------|------|------|
| 创建任务 | 首页右上角「新建任务」 | 填写名称、选配对方式和算法 |
| 修改任务设置 | 任务管理页「⚙️ 设置」按钮 | 可修改名称、描述、配对方式、算法 |
| 重新对比 | 任务管理页「↺ 重新对比」按钮 | 清空历史结果，重新计算全部 |
| 删除任务 | 首页任务卡片「删除」按钮 | 删除任务及所有图片和结果 |

**任务状态**：
- **草稿**：刚创建，还没有图片
- **进行中**：正在计算差异
- **已完成**：所有配对均已计算完毕

### 3.2 图片上传

#### ZIP 批量导入

推荐用于批量上传，ZIP 文件结构要求如下：

```
压缩包.zip
  └── 任意根目录名/
      ├── 原图/          ← 目录名必须是「原图」
      │   ├── img1.png
      │   └── img2.png
      └── 对比图/        ← 目录名必须是「对比图」
          ├── img1.png
          └── img2.png
```

> ⚠️ 图片直接放在 `原图/` 或 `对比图/` 目录下，**不支持再嵌套子目录**

两组图片按**文件名字母顺序**依次配对，数量可以不相等（少的一侧显示"暂无对比图"）。

#### 手动上传与排序

- 直接拖拽图片到上传区，或点击选择
- 上传后可以**拖拽调整排序**（排序决定配对关系）
- 悬停图片卡片右上角出现删除按钮

### 3.4 导出报告

在对比结果页面点击「**↓ 导出报告**」，会下载一个 HTML 文件。

**报告内容**：
- 任务基本信息（名称、总对数、算法）
- 统计概览（总对数 / 正常 / 异常 / 待计算）
- 每对图片的状态列表（点击展开可查看 A/B 原图对比）

**离线使用**：
- 图片总大小 < 50MB：图片直接内嵌在 HTML 中，无需网络即可查看
- 图片总大小 ≥ 50MB：图片通过链接加载，查看时需连接服务（报告内有提示）

> 💡 报告文件可以直接通过邮件/即时通讯发给他人，无需安装任何软件，用浏览器打开即可

### 3.5 对比算法说明

平台提供 4 种对比算法，在创建/修改任务时选择：

| 算法 | 适用场景 | 特点 |
|------|---------|------|
| **标准模式**（默认）| 通用场景，不确定选这个 | 综合评估像素差异和结构变化 |
| **文档对比** | PDF/Word 导出图片对比 | 更严格，能检测 DPI 变化、字体渲染细节 |
| **结构探测** | 检测页面布局、元素增删 | 对宏观结构变化敏感，忽略细节 |
| **像素对比** | 验证两图是否完全一致 | 最严格，几乎不容忍任何差异 |

**如何选择**：
- 一般截图对比 → 标准模式
- PDF/文档导出 → 文档对比
- 页面改版了什么 → 结构探测
- 要验证是否完全一样 → 像素对比

**相似度阈值**：
- ≥ 75% → 标记为「正常」（相似）
- < 75% → 标记为「差异」（显著不同）

### 4.3 修改配置文件（关键步骤，共 2 处）

部署到公网服务器后，需要修改 **2 个配置文件**，否则图片无法正常加载。

#### 修改一：后端环境变量（`backend/.env`）

```bash
nano backend/.env
```

修改 `CORS_ORIGINS` 和新增 `MINIO_PUBLIC_BASE`：

```env
BACKEND_PORT=13011
MINIO_ENDPOINT=localhost:13012
MINIO_ACCESS_KEY=admin
MINIO_SECRET_KEY=请改为更强的密码         # ← 建议修改
MINIO_BUCKET=graph-compare
DATABASE_URL=sqlite:///./graph_compare.db
CORS_ORIGINS=https://你的域名.com          # ← 改为你的域名
MINIO_PUBLIC_BASE=https://你的域名.com/minio  # ← 新增，图片公网访问地址前缀
```

> `MINIO_ENDPOINT` 保持 `localhost:13012`（后端与 MinIO 在同一台机器），图片对外访问通过 `MINIO_PUBLIC_BASE` 走 Nginx 代理。

#### 修改二：图片 URL 生成逻辑（`backend/services/oss_service.py`）

**文件路径**：`backend/services/oss_service.py`
**修改位置**：第 100~106 行，`get_public_url` 函数

```bash
nano backend/services/oss_service.py
```

找到如下内容（第 100 行起）：

```python
def get_public_url(oss_key: Optional[str]) -> Optional[str]:
    if not oss_key:
        return None
    return f"http://{_endpoint}/{BUCKET}/{oss_key}"
```

改为：

```python
def get_public_url(oss_key: Optional[str]) -> Optional[str]:
    if not oss_key:
        return None
    # 优先读 .env 中的 MINIO_PUBLIC_BASE（公网部署时使用）
    # 本地开发时不设置此变量，自动降级为 localhost
    public_base = os.getenv("MINIO_PUBLIC_BASE", f"http://{_endpoint}")
    return f"{public_base}/{BUCKET}/{oss_key}"
```

> **为什么要改这里**：前端展示图片时，图片 URL 由后端 API 返回。本地开发时返回 `http://localhost:13012/...` 没问题，但公网访问时浏览器看不到你服务器的 localhost，需要换成你的域名地址（通过 Nginx 代理到 MinIO）。

### 4.5 配置 Nginx（含前端路由关键配置）

Nginx 承担三个职责：① 托管前端静态文件 ② 转发 API 请求 ③ 转发图片请求到 MinIO

```bash
sudo nano /etc/nginx/sites-available/graph-compare
```

写入以下内容（**替换两处 `你的域名.com`**）：

```nginx
server {
    listen 80;
    server_name 你的域名.com;       # ← 替换为你的域名

    # ── 前端静态文件 ──────────────────────────────────────────
    location / {
        root /root/graph-compare-platform/frontend/dist;
        index index.html;

        # ⚠️  以下这行极其重要，不能省略！
        # React Router 是前端路由，所有子路径（/tasks、/tasks/1/compare 等）
        # 实际上都是同一个 index.html 处理的。
        # 如果没有这行，用户直接访问或刷新子页面时，Nginx 找不到对应文件，返回 404。
        try_files $uri $uri/ /index.html;
    }

    # ── 后端 API 代理 ──────────────────────────────────────────
    location /api/ {
        proxy_pass http://127.0.0.1:13011;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        client_max_body_size 500m;    # 允许最大上传 500MB（ZIP 文件）
        proxy_read_timeout 300s;      # 大文件计算超时
        proxy_connect_timeout 60s;
    }

    # ── MinIO 图片访问代理 ──────────────────────────────────────
    # 前端图片 URL 格式：https://你的域名.com/minio/graph-compare/images/xxx.png
    # 注意：location 末尾的斜杠和 proxy_pass 末尾的斜杠都要有
    location /minio/ {
        proxy_pass http://127.0.0.1:13012/;
        proxy_set_header Host $host;
    }
}
```

启用并验证：

```bash
sudo ln -s /etc/nginx/sites-available/graph-compare /etc/nginx/sites-enabled/
sudo nginx -t          # 必须显示 "syntax is ok"
sudo systemctl reload nginx
```

### 4.6 配置 systemd 开机自启

**MinIO 服务**：

```bash
sudo nano /etc/systemd/system/graph-compare-minio.service
```

```ini
[Unit]
Description=Graph Compare - MinIO
After=network.target

[Service]
Type=simple
User=root
WorkingDirectory=/root/graph-compare-platform
Environment=MINIO_ROOT_USER=admin
Environment=MINIO_ROOT_PASSWORD=请改为和.env相同的密码
ExecStart=/root/graph-compare-platform/bin/minio server /root/graph-compare-platform/minio-data --address ":13012" --console-address ":13013"
Restart=on-failure
RestartSec=5

[Install]
WantedBy=multi-user.target
```

**后端服务**：

```bash
sudo nano /etc/systemd/system/graph-compare-backend.service
```

```ini
[Unit]
Description=Graph Compare - Backend API
After=network.target graph-compare-minio.service

[Service]
Type=simple
User=root
WorkingDirectory=/root/graph-compare-platform/backend
ExecStart=/root/.local/bin/uv run uvicorn main:app --host 0.0.0.0 --port 13011
Restart=on-failure
RestartSec=5

[Install]
WantedBy=multi-user.target
```

启动服务：

```bash
sudo systemctl daemon-reload
sudo systemctl enable graph-compare-minio graph-compare-backend
sudo systemctl start graph-compare-minio
sleep 3
sudo systemctl start graph-compare-backend

sudo systemctl status graph-compare-minio
sudo systemctl status graph-compare-backend
```

## 5. 常见问题 QA

### 安装相关

**Q：运行 `install.sh` 时提示 MinIO 下载失败怎么办？**

A：脚本会自动尝试备用下载源。如果仍然失败，可以手动下载：
- macOS Apple Silicon：https://dl.minio.org.cn/server/minio/release/darwin-arm64/minio
- macOS Intel：https://dl.minio.org.cn/server/minio/release/darwin-amd64/minio
- Linux x86_64：https://dl.minio.org.cn/server/minio/release/linux-amd64/minio

下载后放到项目 `bin/minio` 位置，并执行 `chmod +x bin/minio`。

---

**Q：启动后访问 http://localhost:13010 页面空白怎么办？**

A：依次检查：
1. 运行 `bash scripts/start.sh` 后是否出现"所有服务已启动"
2. 检查日志：`tail -30 logs/backend.log`，看是否有报错
3. 确认端口没有被其他程序占用：`lsof -i :13010`

---

### 上传相关

**Q：ZIP 上传后提示"未找到原图和对比图目录"**

A：检查 ZIP 内部结构是否正确：
- 必须有名为 `原图` 的文件夹和 `对比图` 的文件夹
- 图片直接放在这两个文件夹内，不能再套子文件夹
- Windows 用户注意：确保解压后的目录名确实是"原图"而不是乱码

---

**Q：上传的图片顺序怎么决定配对关系？**

A：同一组内，图片按**上传时的文件名字母顺序**排列，相同位置的图片配对。如果顺序不对，可以在任务管理页面拖拽排序来调整。

---

**Q：两组图片数量不一样可以吗？**

A：可以。数量少的一侧，多出来的位置显示"暂无对比图"，不影响其他配对的计算。

---

### 对比结果相关

**Q：对比结果一直显示"待计算"不更新**

A：可能原因：
1. 图片太大，正在计算中（超大 PDF 截图可能需要几分钟）——查看日志 `tail -f logs/diff.log`
2. 计算出错——在任务管理页点击「重新对比」按钮重新触发

---

**Q：两张明显不同的图片，相似度却显示 80% 以上，怎么回事？**

A：可能是两张图的背景/空白区域占比太大，导致差异被稀释。建议：
1. 切换到「**文档对比**」模式，该模式对内容区域更敏感
2. 或切换到「**像素对比**」模式，获得更严格的判断

---

**Q：相似度分数的含义是什么？75% 是怎么定的？**

A：相似度分数是综合多个维度（像素差异、结构差异、颜色分布）计算出的 0~100% 的值。
- 75% 这个阈值来自实测调优：在 App 截图和 PDF 对比中，75% 能较好地区分"基本一致"和"有明显差异"
- 如果你的场景要求更严格，可以参考右侧面板中的具体分数自行判断

---

**Q：差异图里的红框标注看起来有很多，但实际图片差异不大**

A：红框是基于像素级差异生成的，有时 JPEG 图片的压缩噪声会产生小差异。建议：
1. 使用 PNG 格式导出/上传，减少压缩噪声
2. 切换到「标准模式」（阈值 15），小于 15 的像素差异会被过滤掉

---

### 报告相关

**Q：导出的报告图片加载不出来**

A：报告文件超过 50MB 时，图片使用链接加载（而非内嵌）。此时需要：
1. 确保平台服务仍在运行（`bash scripts/start.sh`）
2. 在服务运行的同一台电脑上打开 HTML 文件

---

**Q：如何把报告分享给没有安装平台的同事？**

A：
- 如果报告 < 50MB（图片已内嵌）：直接发送 HTML 文件，对方用浏览器打开即可，无需任何安装
- 如果报告 ≥ 50MB：需要在服务器部署公网版本，将访问地址分享给对方

---

**Q：服务器部署后，同事访问平台上传图片太慢**

A：原因通常是服务器上传带宽限制。建议：
1. 使用 ZIP 压缩后再上传，减少文件数量
2. 图片分辨率过高时，可以适当降低分辨率（对比结果基本不受影响）

---

*如有其他问题，可查看 `logs/access.log`（HTTP 请求日志）和 `logs/backend.log`（后端日志）获取详细信息。*
